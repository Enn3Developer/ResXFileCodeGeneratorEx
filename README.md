# ResXFileCodeGeneratorEx for Rider and ReSharper

[![ReSharper](https://img.shields.io/jetbrains/plugin/v/RESHARPER_PLUGIN_ID.svg?label=ReSharper&colorB=0A7BBB&style=for-the-badge&logo=resharper)](https://plugins.jetbrains.com/plugin/RESHARPER_PLUGIN_ID)

A Rider/ReSharper port of the classic **ResXFileCodeGeneratorEx** Visual Studio custom tool.
When set as the *Custom Tool* for a `.resx` file, it generates a `.Designer.cs` companion class
with:

- strongly-typed static properties for every resource entry,
- `XxxFormat(object arg0, ŌĆ”)` helper methods for resource strings containing `{0}`, `{1}`, ŌĆ”
  placeholders,
- a nested `ResourceNames` class exposing each resource key as a constant,
- a thread-safe `ResourceManager` and an overridable `Culture` property,
- XML doc-comments describing each member.

## Visual Studio compatibility

The generated `.Designer.cs` is intentionally kept **byte-for-byte compatible** with the output of
the original Visual Studio tool (DMK Software's `StronglyTypedResourceBuilderEx`, rendered through
CodeDom). This means a team where some developers use Rider and others use Visual Studio can
regenerate the same `.resx` without producing spurious diffs or compile errors. The only lines that
can legitimately differ are the copyright year in the header comment (which tracks the current year,
exactly as the VS tool does).

This plugin is a port of **[alejack9/ResXFileCodeGeneratorEx](https://github.com/alejack9/ResXFileCodeGeneratorEx)**

Dmytro Kryvko / DMK Software). All credit for the original generator and its output format belongs to
those authors; this repository only adapts it to the Rider/ReSharper backend.
